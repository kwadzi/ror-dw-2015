what's a webapp anyway?

A web application is an application utilizing web and [web] browser technologies to accomplish one or more tasks over a network, typically through a [web] browser.

-Jarel Remick, AppStorm

let's talk about Rails

Ruby
on Rails

  • Open Source (Source code)
  • Released: 2003
  • Full stack web framework written in Ruby

full stack... what?

it abstracts and manages all parts of a web application:

  • the database (model)
  • HTML/visualization (view)
  • request flow (controller)

MVC

model

  • maps a table in the database with a ruby object
  • validates your data before saving them
  • here you manipulate your data
  • here you add associations

controller

  • each method is a function that your application exposes
  • controls the flow of the request
  • exposes your data to views

views

  • show data to the user (in various format: html, csv, pdf..)

Rails philosophy

  • Agile development
  • Ruby => less and more readable code, shorter development times, simple but powerful, no compilation
  • MVC, DRY, Testing
  • convention over configuration
  • SQL and JavaScript are abstracted
  • AJAX and REST
  • excellent community, tools, and documentation

convention over configuration

Rails uses standard names and a standard directory structure for every webapp 

              app
              |-- assets
              |-- controllers
              |-- helpers
              |-- mailers
              |-- models
              `-- views

for example

if we have a model called Book, Rails will assume the following directory structure


              app
              |-- controllers
              |   `-- books_controller.rb
              |-- models
              |   `-- book.rb
              `-- views
                  `-- books
                      |-- index.html.erb
                      |-- new.html.erb
                      |-- edit.html.erb
                      `-- ...

and it will know that there is a table called books in the DB

views are rendered automatically

in our example, books_controller will automatically render views if they are named correctly 

              class BooksController < ActiveController
                def index
                  # renders app/views/books/index.html.erb
                  # no need to write anything here!
                end

                def new
                  # renders app/views/books/new.html.erb
                  # and I don't need to write anything here!
                end
              end
