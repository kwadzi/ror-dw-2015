Ruby on Rails

is a framework for building web applications

  • Open Source (Source code)
  • Released: 2003
  • Full stack web framework written in Ruby

full stack... what?

it abstracts and manages all parts of a web application:

  • the database (model)
  • HTML/visualization (view)
  • request flow (controller)

ModelVC

Models are classes that talk to the databse.
You find, create and save models, so you donâ€™t (usually) have to write SQL. Rails has a class to handle the magic of saving to a database when a model is updated.

A Rails model

  • maps a table in the database with a ruby object
  • validates your data before saving them
  • has the code manipulate your data
  • manages the associations with other models

MVController

Controllers take user input (like a URL) and decide what to do (show a page, order an item, post a comment).
They may have business logic, like finding the right models or changing data. Ideally, controllers just take inputs, call model methods, and pass outputs to the view (including error messages).

A Rails controller

  • has methods that correspond to the app's functions
  • controls the flow of the request
  • exposes your data to views

MViewC

Views display the output to the user (typically in HTML)

A Rails view

  • uses ERB to put Ruby code in HTML
  • can output data XML (for web services/RSS feeds) or JSON (for AJAX calls)

MVC

convention over configuration

Rails uses standard names and a standard directory structure for every webapp 

              app
              |-- assets
              |-- controllers
              |-- helpers
              |-- mailers
              |-- models
              `-- views

for example

if we have a model called Book, Rails will assume the following directory structure


              app
              |-- controllers
              |   `-- books_controller.rb
              |-- models
              |   `-- book.rb
              `-- views
                  `-- books
                      |-- index.html.erb
                      |-- new.html.erb
                      |-- edit.html.erb
                      `-- ...

and it will know that there is a table called books in the DB

views are rendered automatically

in our example, books_controller will automatically render views if they are named correctly
